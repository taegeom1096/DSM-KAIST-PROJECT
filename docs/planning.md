## 서비스 이름

## 한 줄 설명
- 이 서비스는 굶주린 사용자가 배고픈 상황에서 메뉴 결정을 할 수 있게 돕는다.

## 브레인스토밍
원클릭 배고픔 폭탄 (가장 쉬움): 화면에 불타는 붉은색 [배고파 뒤지겠다] 버튼 딱 하나만 둡니다. 누르면 0.1초 만에 삼겹살, 엽떡 같은 고칼로리 음식을 화면 가득 큰 글씨로 쾅 찍어줍니다.

로딩 스피너 (슬롯머신): 버튼을 누르면 화면의 글자가 피자 ➡️ 치킨 ➡️ 족발 ➡️ 국밥 순으로 1초 동안 미친 듯이 바뀌다가 쾅! 하고 최종 메뉴 하나에 멈추는 시각 효과입니다. (자바스크립트 setInterval 함수만 쓰면 끝납니다.)

이상형 월드컵 (토너먼트 랜덤): DB 없이 메뉴 4개(짜장 vs 짬뽕, 피자 vs 치킨)를 랜덤으로 화면에 띄우고 사용자가 고르게 만듭니다. 최종 우승 메뉴를 보여주는 방식입니다.
### 아이디어 1
- 사용자:선택장애가 심하게 와서 뇌가 정지된 배고픈 사람
- 상황:점심/저녁 메뉴를 고르느라 30분째 배달 앱만 붙잡고 침 흘리며 고통받고 있을 때
- 액션: 결정장애 유저가 배고파 뒤지겠다 버튼을 클릭하여 0.1초 만에 딱 하나의 음식을 강제 강탈(강제 추천)한다.

### 아이디어 2
- 사용자:메뉴 고르는 것조차 귀찮고 재미를 원하는 자취생 또는 대학생
- 상황: 매일 먹는 밥이 거기서 거기라 무난한 메뉴를 조금 더 재밌게 뽑고 싶을 때
- 액션: 지친 사용자가 화면의 긁는 복권(스크래치)을 마우스로 쓱쓱 긁어서 숨겨진 오늘의 메뉴를 확인한다.

### 아이디어 3
- 사용자:돈은 없고 배는 고픈데 집에 재료는 몇 개 남은 자취러
- 상황:배달비 아깝고, 냉장고에 썩어가는 재료(계란, 양파 등)로 대충 한 끼 때우고 싶을 때
- 액션:돈 없는 자취생이 집에 있는 남은 재료 태그를 선택하여 만들 수 있는 최적의 메뉴를 랜덤 도출한다.

## 액션 작성 규칙
- 좋은 예: 학생이 모집 글을 작성한다
- 좋은 예: 작성자가 모집 글을 수정한다
- 좋은 예: 학생이 신청 댓글을 삭제한다
- 아쉬운 예: 글 작성
- 아쉬운 예: 목록 보기
